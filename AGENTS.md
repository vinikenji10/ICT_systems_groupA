# SIT Club Hub — Agent Guide

## Project structure

```
public/              Static site (vanilla HTML/CSS/JS, no build step)
  index.html         Main layout — navbar, sidebar filter panel, club card grid, footer
  css/style.css      Full design system (colors, typography, responsive breakpoints)
  js/app.js          Club dataset (10 clubs), filter logic, search, mobile drawer toggle
 404.html            Branded 404 page
functions/           Python Cloud Functions (Firebase, Python 3.13)
  main.py            Stub only — no deployed functions yet
  requirements.txt   Pinned deps (firebase-functions 0.5.0, firebase-admin 7.4.0)
firebase.json        Hosting→`public/`, Functions→`functions/`, Firestore→asia-northeast1
.firebaserc          Default project: `ict-systems-project-a`
firestore.rules      Dev only — wide open, expires 2026-05-22 (update before then)
DESIGN.md            Design system spec (colors, typography, layout, components)
```

## Commands

| Action | Command |
|---|---|
| Preview locally | `python3 -m http.server 8080 -d public` |
| Deploy all | `firebase deploy` |
| Deploy hosting only | `firebase deploy --only hosting` |
| Deploy functions only | `firebase deploy --only functions` |

No `package.json`, no npm install, no build step for the frontend.

## CI quirks

`firebase-hosting-pull-request.yml` runs `npm ci && npm run build` but there is no `package.json` in the repo — the CI is **broken** as auto-generated by Firebase. Either remove the `run` step or add a stub `package.json`.

## Code conventions

- All club data lives in `public/js/app.js` as a JS array — no JSON file, no Firestore seed.
- The 4 typos called out in DESIGN.md §5 (SIT Esports misspelling, garbled descriptions) were **already fixed** in `app.js`.
- Design tokens are defined in `DESIGN.md` and implemented in `style.css` — colors are `#0C4A34` (primary), `#F4F1EA` (bg), `#E6D7C3` (accent tag).
- Club cards are rendered client-side by `renderClubs()` in `app.js`. Filtering/search is all in-memory (no API calls).
- Mobile sidebar uses a `.sidebar-open` body class toggle — no JS framework.

## What's missing / known gaps

- No test framework or linter configured anywhere.
- Python functions in `functions/main.py` are a stub — `initialize_app()` is commented out, no HTTP handlers registered.
- Firestore is provisioned but has no collections/data — no seed script exists.
- The CI has no lint/typecheck/test step, would benefit from one.
